3943. Number of Pairs After Increment

You are given two integer arrays `nums1` and `nums2`, and a 2D integer array `queries`.

Each `queries[i]` is one of the following types:

`[1, x, y, val]` – Add `val` to every element in `nums2[x..y]`.
`[2, tot]` – Compute the number of pairs `(j, k)` such that `nums1[j] + nums2[k] == tot`.

Return an integer array `answer`, where `answer[j]` is the number of pairs for the `j`th query of type 2.

 

**Example 1:**
```
Input: nums1 = [1,2], nums2 = [3,4], queries = [[2,5],[1,0,0,2],[2,5]]

Output: [2,1]

Explanation:

queries[0] = [2, 5]: Valid pairs are nums1[0] + nums2[1] = 1 + 4 = 5 and nums1[1] + nums2[0] = 2 + 3 = 5.
queries[1] = [1, 0, 0, 2]: Add 2 to nums2[0], resulting in nums2 = [5, 4].
queries[2] = [2, 5]: Valid pair is nums1[0] + nums2[1] = 1 + 4 = 5.
Thus, the answer = [2, 1].
```

**Example 2:**
```
Input: nums1 = [1,1], nums2 = [2,2,3], queries = [[2,4],[1,0,1,1],[2,4]]

Output: [2,6]

Explanation:

queries[0] = [2, 4]: Valid pairs are nums1[0] + nums2[2] = 1 + 3 and nums1[1] + nums2[2] = 1 + 3.
queries[1] = [1, 0, 1, 1]: Add 1 to nums2[0] and nums2[1], resulting in nums2 = [3, 3, 3].
queries[2] = [2, 4]: Every element of nums1 = [1, 1] pairs with every element of nums2 = [3, 3, 3] as 1 + 3 = 4. That gives 2 × 3 = 6 pairs in total.
Thus, the answer = [2, 6].
```

**Example 3:**
```
Input: nums1 = [2,5,8,4], nums2 = [1,3,8], queries = [[2,9],[1,1,2,1],[2,10]]

Output: [1,0]

Explanation:

queries[0] = [2, 9]: Only valid pair is nums1[2] + nums2[0] = 8 + 1 = 9.
queries[1] = [1, 1, 2, 1]: Add 1 to nums2[1] and nums2[2], resulting in​​​​​​​ nums2 = [1, 4, 9].
queries[2] = [2, 10]: No pair sums to 10.
Thus, the answer = [1, 0].
```

**Constraints:**

* `1 <= nums1.length <= 5`
* `1 <= nums2.length <= 5 * 10^4`
* `1 <= nums1[i], nums2[i] <= 10^5`
* `1 <= queries.length <= 5 * 10^4`
* `queries[i].length == 2 or 4`
    * `queries[i] == [1, x, y, val], or`
    * `queries[i] == [2, tot]`
    * `0 <= x <= y < nums2.length`
    * `1 <= val <= 10^5`
    * `1 <= tot <= 10^9`

**Solution 1: (SQRT Decomposition)**

1. Intuition
The problem tasks me with handling two types of operations: range additions on nums2 and counting pairs from nums1 and nums2 that sum up to a target tot. Crucially, I noticed the constraint on nums1: its length is extremely small (1≤nums1.length≤5). This is the core hint.

Because nums1 is nearly constant in size, calculating pairs that satisfy nums1[j]+nums2[k]==tot can be reframed as iterating through the few elements of nums1 and looking up the frequency of the required matching value (tot−nums1[j]) inside nums2.

To handle the range updates and frequency lookups efficiently on nums2, my approach uses SQRT Decomposition. By dividing nums2 into blocks of size  
N
​
 , I can update an entire block in O(1) time using a lazy tag (help2), and partial blocks can be updated manually in O( 
N
​
 ) time. Lookups check each block's frequency map while factoring in its active lazy tag.

2. Algorithm
Step 1: Decomposition Setup
Split nums2 into segments of length block= 
N
​
 .
Create a vector of hash maps help where help[b][val] stores the frequency of val in block b.
Maintain a lazy array help2[b] to record cumulative values added to block b as a whole.
Step 2: Range Update Operations (Type 1)
When adding val to nums2[x..y]:

Partial Blocks: If the range covers only elements inside a block partially (at the left or right boundaries), I manually decrement the frequency of the old value, increment the new value inside help, and update the scalar nums[i] array in O( 
N
​
 ) time.
Full Blocks: For blocks completely covered by the range, instead of iterating over each element, I add val to the lazy accumulator help2[b] in O(1) time.
Step 3: Pair Counting Queries (Type 2)
When calculating pairs for a target tot:

Loop through each element num inside nums1.
The needed value from nums2 is target = tot - num.
To find its total count in nums2, query each block b:
Due to the lazy tag, a physical value X inside block b currently represents the value X+help2[b].
Therefore, to find elements equal to target, I look up target - help2[b] inside the block's frequency map: help[b][target - help2[b]].
Accumulate the matches across all blocks and append the result to ans.

4. Implementation Summary
The combination of a hyper-constrained nums1 array and SQRT Decomposition on nums2 provides a balanced trade-off. Instead of a heavier segment tree with complex lazy propagation, splitting the tracking arrays into

class Solution {
public:
    vector<int> numberOfPairs(vector<int>& nums1, vector<int>& nums2, vector<vector<int>>& queries) {
        int n=nums2.size();
        int block=sqrt(n);
        int sz=block;
        vector<long long> nums(n);
        
        
        if(block*block<n)
            sz+=2;
       
        vector<unordered_map<long long,int>> help(sz);
        vector<long long> help2(sz,0);

        for(int i=0;i<n;i++)
            {
                help[i/block][nums2[i]]++;
                nums[i]=nums2[i];
            }

        auto update=[&](int i,int j,long long val)
        {
            if(j-i+1<=2*block)
            {
                while(i <=j )
                {
                    help[i/block][nums[i]]--;
                    help[i/block][nums[i]+val]++;
                    nums[i]=nums[i]+val;
                    i++;
                }
                return ;
                
            }
            while(i%block !=0 )
                {
                    help[i/block][nums[i]]--;
                    help[i/block][nums[i]+val]++;
                    nums[i]=nums[i]+val;
                    i++;
                }
            while(j%block != block-1)
                {
                    help[j/block][nums[j]]--;
                    help[j/block][nums[j]+val]++;
                    nums[j]=nums[j]+val;
                    j--;
                }
            while((i/block)<=(j/block))
                {
                    help2[i/block]+=val;
                    i+=block;
                }
            return ;
        };

        auto queryAns=[&](int val){
            int ans=0;
            int i=0;
            while(i<sz)
                {
                    ans+=help[i][val-help2[i]];
                    i++;
                }
            return ans;
            
        };
        vector<int> ans;

        for(auto &query:queries)
            {
                if(query[0]==1)
                {
                    int i=query[1],j=query[2],val=query[3];

                    if(n>100)
                    update(i,j,val);
                    else{
                        while(i <=j )
                            {
                                help[i/block][nums[i]]--;
                                help[i/block][nums[i]+val]++;
                                nums[i]=nums[i]+val;
                                i++;
                            }
                        
                    }
                }
                else{
                    int anscur=0;
                    for(auto num:nums1)
                        {
                            anscur+=queryAns(query[1]-num);
                            
                        }
                    ans.push_back(anscur);
                    
                }
                
            }
        return ans;
    }
};
