You are given two integers l and r.

An integer is called **good** if its digits form a **strictly** monotone sequence, meaning the digits are **strictly increasing** or **strictly decreasing**. All single-digit integers are considered **good**.

An integer is called **fancy** if it is good, or if the **sum of its digits** is good.

Return an integer representing the number of fancy integers in the range `[l, r]` (inclusive).

A sequence is said to be **strictly increasing** if each element is **strictly greater** than its previous one (if exists).

A sequence is said to be **strictly decreasing** if each element is **strictly less** than its previous one (if exists).

 

**Example 1:**
```
Input: l = 8, r = 10

Output: 3

Explanation:

8 and 9 are single-digit integers, so they are good and therefore fancy.
10 has digits [1, 0], which form a strictly decreasing sequence, so 10 is good and thus fancy.
Therefore, the answer is 3.
```

**Example 2:**
```
Input: l = 12340, r = 12341

Output: 1

Explanation:

12340
12340 is not good because [1, 2, 3, 4, 0] is not strictly monotone.
The digit sum is 1 + 2 + 3 + 4 + 0 = 10.
10 is good as it has digits [1, 0], which is strictly decreasing. Therefore, 12340 is fancy.
12341
12341 is not good because [1, 2, 3, 4, 1] is not strictly monotone.
The digit sum is 1 + 2 + 3 + 4 + 1 = 11.
11 is not good as it has digits [1, 1], which is not strictly monotone. Therefore, 12341 is not fancy.
Therefore, the answer is 1.
```

**Example 3:**
```
Input: l = 123456788, r = 123456788

Output: 0

Explanation:

123456788 is not good because its digits are not strictly monotone.
The digit sum is 1 + 2 + 3 + 4 + 5 + 6 + 7 + 8 + 8 = 44.
44 is not good as it has digits [4, 4], which is not strictly monotone. Therefore, 123456788 is not fancy.
Therefore, the answer is 0.
```
 

**Constraints:**

`1 <= l <= r <= 10^15`

# Submissions
---
**Solution 1: (Digit DP)**

**Intuition**
So bro, the problem is basically asking — how many numbers between l and r are "fancy"?
A number is fancy if:

its digits go strictly up (like 1 3 5 7), OR
its digits go strictly down (like 9 7 4 2), OR
if neither of those, then its digit sum itself should have strictly increasing or decreasing digits

The moment I see "count numbers in a range [l, r] with some digit property" — that's a classic Digit DP signal. And since we need f(r) - f(l-1), we just run our DP twice.

**Approach**
We do digit DP with these states:

i → which digit position we're at
prev → what digit we placed last
dir → direction so far: 0 = just started (first real digit), 1 = strictly increasing, 2 = strictly decreasing, 3 = neither (gone sideways already)
tight → are we still bounded by the original number? (classic digit DP thing)
lz → leading zeros flag (so 007 isn't treated as a 3-digit number)
sum → digit sum so far

At the end (base case), we decide:

If dir is 1 or 2 → fancy! ✅
If dir is 0 → single digit number, always fancy ✅
If dir is 3 (digits went neither way) → check if the digit sum itself is fancy using isgood() ✅

The dir transitions are straightforward — once you go "neither" (dir == 3), you're stuck there. Otherwise you update based on whether the new digit is bigger or smaller than previous.
The sum dimension goes up to 199 max (18 nines = 162, so 200 is safe).

```c++
class Solution {
    long long dp[16][11][4][2][2][200];

    bool isgood(int sum){
        string s = to_string(sum);

        if(s.length() == 1) return true;

        bool inc = true;
        bool dec = true;

        for(int i = 1 ; i < s.length() ; i++){
            if(s[i] <= s[i-1]) inc = false;
            if(s[i] >= s[i-1]) dec = false;
        }

        return inc || dec;
    }

    long long solve(int i , int prev , int dir , int tight , int lz , int sum, string &s){
        if(i == s.length()){
            if(lz == 0) return 0;

            if(dir == 1 || dir == 2 || dir == 0) return 1;

            if(isgood(sum)) return 1;

            return 0;
        }

        int li = prev + 1;
        if(dp[i][li][dir][tight][lz][sum] != -1){
            return dp[i][li][dir][tight][lz][sum];
        }

        int ub = (tight == 1)? s[i] - '0' : 9 ;
        long long ans = 0;

        for(int d = 0 ; d <= ub ; d++){
            int nt = (tight == 1 && d == ub)? 1:0;

            if(lz == 0 && d == 0){
                ans += solve(i + 1 , -1 , 0 , nt , 0 , sum , s);
                continue;
            }else{
                int ns = sum + d;

                if(lz == 0){
                    ans += solve(i + 1 , d , 0 , nt , 1 , ns , s);
                    continue;
                }

                int nd;
                if(dir == 3){
                    nd = 3;
                }else if(dir == 0){
                    if(d > prev ) nd = 1;
                    else if(d < prev) nd = 2;
                    else nd = 3;
                }else if(dir == 1){
                    nd = (d > prev)? 1 : 3;
                }else{
                    nd = (d < prev)? 2 : 3;
                }

                ans += solve(i + 1 , d , nd , nt , 1 , ns , s);
            }
        }

        return dp[i][li][dir][tight][lz][sum] = ans;
    }
public:
    long long countFancy(long long l, long long r) {
        string left = to_string(l - 1);
        string right = to_string(r);

        memset(dp , -1 , sizeof(dp));
        long long a = solve(0 , -1 , 0 , 1 , 0 , 0 , right);

        memset(dp , -1 , sizeof(dp));
        long long b = solve(0 , -1 , 0 , 1 , 0 , 0 , left);

        return a - b;
    }
};
```
