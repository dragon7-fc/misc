1340. Jump Game V

Given an array of integers `arr` and an integer `d`. In one step you can jump from index `i` to index:

* `i + x` where: `i + x < arr.length` and `0 < x <= d`.
* `i - x` where: `i - x >= 0` and `0 < x <= d`.

In addition, you can only jump from index `i` to index `j` if `arr[i] > arr[j]` and `arr[i] > arr[k]` for all indices `k` between `i` and `j` (More formally `min(i, j) < k < max(i, j)`).

You can choose any index of the array and start jumping. Return the maximum number of indices you can visit.

Notice that you can not jump outside of the array at any time.

 

**Example 1:**

![1344_meta-chart.jpeg](img/1344_meta-chart.jpeg)
```
Input: arr = [6,4,14,6,8,13,9,7,10,6,12], d = 2
Output: 4
Explanation: You can start at index 10. You can jump 10 --> 8 --> 6 --> 7 as shown.
Note that if you start at index 6 you can only jump to index 7. You cannot jump to index 5 because 13 > 9. You cannot jump to index 4 because index 5 is between index 4 and 6 and 13 > 9.
Similarly You cannot jump from index 3 to index 2 or index 1.
```

**Example 2:**
```
Input: arr = [3,3,3,3,3], d = 3
Output: 1
Explanation: You can start at any index. You always cannot jump to any index.
```

**Example 3:**
```
Input: arr = [7,6,5,4,3,2,1], d = 1
Output: 7
Explanation: Start at index 0. You can visit all the indicies. 
```

**Example 4:**
```
Input: arr = [7,1,7,1,7,1], d = 2
Output: 2
```

**Example 5:**
```
Input: arr = [66], d = 1
Output: 1
```

**Constraints:**

* `1 <= arr.length <= 1000`
* `1 <= arr[i] <= 10^5`
* `1 <= d <= arr.length`

**Solution 1: (DP, TOP-down)**

For each step `A[i]`,
check `A[j]` on its left and right,
until it meet the bound or meet the bigger step.

* Time complexity O(ND)
* Space complexity O(N) for dp

```
Runtime: 720 ms
Memory Usage: 17.2 MB
```
```python
class Solution:
    def maxJumps(self, arr: List[int], d: int) -> int:
        N = len(arr)
        res = [0] * N

        @functools.lru_cache(None)
        def dp(i):
            res[i] = 1
            for di in [-1, 1]:
                for j in range(i + di, i + d*di + di, di):
                    if not (0 <= j < N and arr[j] < arr[i]): break
                    res[i] = max(res[i], dp(j) + 1)
            return res[i]

        return max(map(dp, range(N)))
```

**Solution 2: (DP, Bottom-up)**

We can only jump lower, and one step needs the result from its lower step.
So we sort `A[i]` do the dp starting from the smallest.

For each `A[i]`, we check the lower step on the left and right.
This process is O(D) on both side.

* Time complexity O(NlogN + ND), where we are given D <= N
* Space complexity O(N) for dp

```
Runtime: 664 ms
Memory Usage: 12.9 MB
```
```python
class Solution:
    def maxJumps(self, arr: List[int], d: int) -> int:
        N = len(arr)
        dp = [1] * N
        for a, i in sorted([a, i] for i, a in enumerate(arr)):
            for di in [-1, 1]:
                for j in range(i + di, i + d*di + di, di):
                    if not (0 <= j < N and arr[j] < arr[i]): break
                    dp[i] = max(dp[i], dp[j] + 1)
        return max(dp)
```

```c++
class Solution {
    vector<int> memo;
    int n;

    int dfs(int i, const vector<int>& arr, int d) {
        // If already computed, return the cached result
        if (memo[i] != 0) {
            return memo[i];
        }

        int max_visited = 1; // You can always at least visit the current index i

        // 1. Look to the right (i + x)
        for (int x = 1; x <= d && i + x < n; ++x) {
            int j = i + x;
            // Line of sight block: if we hit a bar >= current bar, we can't jump here or further
            if (arr[j] >= arr[i]) {
                break;
            }
            max_visited = max(max_visited, 1 + dfs(j, arr, d));
        }

        // 2. Look to the left (i - x)
        for (int x = 1; x <= d && i - x >= 0; ++x) {
            int j = i - x;
            // Line of sight block
            if (arr[j] >= arr[i]) {
                break;
            }
            max_visited = max(max_visited, 1 + dfs(j, arr, d));
        }

        return memo[i] = max_visited;
    }
public:
    int maxJumps(vector<int>& arr, int d) {
        n = arr.size();
        memo.assign(n, 0);
        
        int total_max = 0;
        // Try starting from every possible index in the array
        for (int i = 0; i < n; ++i) {
            total_max = max(total_max, dfs(i, arr, d));
        }
        
        return total_max;
    }
};
```

**Solution 3: (DP Bottom-Up, sort)**

DP Top-Down (On-Demand Execution)
DP Bottom-Up (Pre-Planned Execution)

DP Bottom-Up need sort because
Bottom-Up does not use recursion or a call stack. It uses a flat, iterative loop. It assumes that when the loop arrives at index i, all the dependencies (j) have already been calculated and are sitting quietly in the dp array

```c++
class Solution {
public:
    int maxJumps(vector<int>& arr, int d) {
        int n = arr.size();
        vector<int> dp(n, 1); // Every index can at least visit itself
        
        // Step 1: Collect value-index pairs
        vector<pair<int, int>> sortedArr;
        sortedArr.reserve(n);
        for (int i = 0; i < n; ++i) {
            sortedArr.push_back({arr[i], i});
        }
        
        // Step 2: Sort by value ascending
        sort(sortedArr.begin(), sortedArr.end());
        
        int globalMax = 1;
        
        // Step 3: Bottom-up DP processing from smallest to largest
        for (const auto& [val, i] : sortedArr) {
            
            // Look right
            for (int x = 1; x <= d && i + x < n; ++x) {
                int j = i + x;
                if (arr[j] >= val) break; // Line of sight blocked
                dp[i] = max(dp[i], 1 + dp[j]);
            }
            
            // Look left
            for (int x = 1; x <= d && i - x >= 0; ++x) {
                int j = i - x;
                if (arr[j] >= val) break; // Line of sight blocked
                dp[i] = max(dp[i], 1 + dp[j]);
            }
            
            globalMax = max(globalMax, dp[i]);
        }
        
        return globalMax;
    }
};
```
